## Build Issues Encountered & Resolved

### 1. `minimp4` crate required libclang (C dependency)
**Problem:** `minimp4` wraps a C library via bindgen, needs `libclang.dll`.
**Fix:** Replaced with `muxide` — pure Rust MP4 muxer with AV1 support, zero C deps.

### 2. `rav1e` needed NASM for assembly
**Problem:** rav1e's `asm` feature (default) requires NASM which wasn't installed.
**Fix:** Disabled default features, kept only `threading`. Pure Rust fallback works but encoding is slow (~220s for 30 frames).

### 3. rav1e `Frame::planes[n].p()` API mismatch
**Problem:** The Phase 1 plan used `*y_plane.p(col, row) = y` but `p()` returns a value copy, not `&mut`.
**Fix:** Used `data_origin_mut()` with stride-based indexing instead.

### 4. Raw string delimiter collisions with `#` in hex colors
**Problem:** `r#"..."#` raw strings broke when JSON contained `"#000000"` (the `"#` ended the raw string).
**Fix:** Used `r##"..."##` double-hash delimiters for all test JSON containing hex colors.

### 5. Skia ICU linker error on Windows
**Problem:** `skunicode_icu.lib` referenced Windows registry APIs (`RegOpenKeyExW` etc) not linked.
**Fix:** Added `build.rs` that links `advapi32.lib` on Windows.

### 6. Skia `from_raster_data` deprecated
**Problem:** `Image::from_raster_data()` deprecated in skia-safe 0.82.
**Fix:** Replaced with `images::raster_from_data()`.

### 7. muxide AV1 sequence header requirement
**Problem:** muxide requires first AV1 frame to contain a Sequence Header OBU.
**Fix:** Pass entire first rav1e packet as sequence header + fallback to IVF if muxing fails.

## CLI Usage

```bash
# Validate
mmot validate video.mmot.json

# Render to MP4
mmot render video.mmot.json -o output.mp4

# Render with props
mmot render template.mmot.json --prop title="Hello" --prop color="#FF0000" -o output.mp4

# Render with verbose progress
mmot render video.mmot.json -o output.mp4 -v
```
